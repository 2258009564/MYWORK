## 什么是静态变量/成员变量/局部变量？有何性质？
- **成员变量**：定义在类中、方法外。随对象创建而存在，有默认初始值（如 `int` 为 0）。
- **静态变量**：被 `static` 修饰的成员变量。属于类，全类共享一份，随类加载而加载。
- **局部变量**：定义在方法或代码块内。必须先手动初始化才能使用，方法执行完即销毁。

## 什么是overload和override？有什么区别？
- **Overload（重载）**：发生在同一个类中。方法名相同，但参数列表（个数、类型、顺序）不同。
- **Override（重写）**：发生在父子类继承关系中。子类覆盖父类同名同参数的方法，以提供特定实现。

## public、protected、default、private成员变量的访问权限

|**修饰符**|**本类**|**同包**|**子类**|**外部包**|
|---|---|---|---|---|
|**public**|✅|✅|✅|✅|
|**protected**|✅|✅|✅|❌|
|**default** (不写)|✅|✅|❌|❌|
|**private**|✅|❌|❌|❌|

## 在异常处理的过程中，try,catch和finally各有什么作用。
- **try**：用于包裹可能抛出异常的代码段。
- **catch**：当 `try` 中发生异常时，捕获对应类型的异常并进行处理。
- **finally**：无论是否发生异常，其中的代码一定会执行（通常用于资源释放）。

## 什么是继承/封装/多态？
- **封装**：将数据和操作隐藏在类内部，通过 `private` 和 `getter/setter` 控制访问，保护数据。
- **继承**：子类继承父类的属性和方法，实现代码复用（`extends`）。
- **多态**：父类引用指向子类对象，同一个接口在不同实例下表现出不同行为。

## 请解释this/super关键字的使用方法。
- **this**：指代当前对象本身。用于解决成员变量与局部变量重名问题，或调用本类构造器。
- **super**：指代父类空间。用于访问父类的成员变量、成员方法或调用父类构造器。

## 说明抽象类和接口有什么区别。
- **抽象类**（abstract class）：可以有构造方法、普通成员变量，描述“是什么”（is-a）。单继承。
- **接口**（interface）：主要定义行为规范，描述“能做什么”（has-a）。支持多实现。

## final关键字的作用。
- 修饰**类**：该类不可被继承。
- 修饰**方法**：该方法不可被子类重写。
- 修饰**变量**：该变量变为常量，一旦赋值不可更改。

## 实现多线程的方法。
1. 继承 `Thread` 类。
2. 实现 `Runnable` 接口（更常用，解耦）。
3. 实现 `Callable` 接口（可获取返回值）。
4. 通过线程池（ExecutorService）。

## 列举Java中常用的字节输入流和字节输出流并说明其特点，至少3对。
1. **FileInputStream / FileOutputStream**：直接操作文件，性能基础。
2. **BufferedInputStream / BufferedOutputStream**：带缓冲区，减少磁盘 IO 次数，效率高。
3. **ObjectInputStream / ObjectOutputStream**：用于对象的序列化与反序列化。

## 多线程中start方法和run方法分别有什么作用？
- **start()**：启动新线程的方法。调用后，系统会开启新线程并自动执行 `run()` 里的任务。
- **run()**：封装具体执行的任务代码。若直接调用 `run()`，它只是普通的方法调用，不会开启新线程。

## 简述Java中的异常处理机制的简单原理和应用。
- **原理**：当错误发生时，JVM 会创建一个异常对象并抛出。系统会根据 `try-catch` 寻找处理者；若无处理者，异常逐层向上抛给调用者，直至 JVM 报错终止。
- **应用**：通过捕获异常避免程序崩溃，并在发生错误时执行补救措施（如记录日志或回滚事务）。

## 简述throw和throws关键字的区别。
- **throw**：在方法内部使用，手动抛出一个具体的异常对象。
- **throws**：在方法签名处使用，声明该方法可能抛出的异常类型，将处理责任交给调用者。

## 简述静态方法与非静态方法的区别？
- **静态方法**：属于类，用类名直接访问。不能直接访问非静态成员，因为执行时不一定有对象。
- **非静态方法**：属于对象，必须先创建对象才能访问。可以访问类中的所有成员（静态和非静态）。

## 简述实现字节流/字符流的读写过程。
1. 创建流对象（绑定源或目标文件）。
2. 循环读/写数据（通过字节数组 `byte[]` 或字符数组 `char[]` 作为中转）。
3. 刷新缓冲区（字符流必须）。
4. 释放资源（`close`）。

## 什么叫向上转型/向下转型？
- **向上转型**：子类引用的对象赋值给父类引用（如 `Animal a = new Cat();`）。自动转换，安全。
- **向下转型**：父类引用强制转换为子类引用（如 `Cat c = (Cat)a;`）。需配合 `instanceof` 判断以防报错。

## 什么叫泛型？
**泛型**是一种“标签”或“参数化类型”。在定义类或方法时使用占位符（如 `<T>`），直到实际使用时才确定具体类型。作用是保证类型安全，避免在运行时出现 `ClassCastException`。
